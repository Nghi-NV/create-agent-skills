# Documentation Standards

## Principles
1.  **Visuals First**: Always start with a Mermaid diagram for flows/architecture.
2.  **Searchability**: Use action-oriented headings (e.g., "Integrating X" vs "Integration").
3.  **Copy-Pasteable**: All code blocks must be complete and runnable.

## Mermaid Guidelines
-   Use `graph TD` for hierarchies.
-   Use `sequenceDiagram` for API calls.
-   Keep diagrams simple (max 10 nodes).
